### Clinical Records

Status: Completed – Sprint 2.2.1

The Medical Treatment page has been modernized as the central Clinical Records module.

Completed:

- Standard Govardhana page header and layout.
- Summary cards:
  - Total Records
  - Selected Period
  - Today's Cases
  - Cattle Treated
  - Doctors
- From Date and To Date filtering.
- Search by cattle ID, disease, medicine and doctor.
- Disease/Symptom and Doctor filters.
- Click anywhere on a row to open Clinical Record Details.
- Separate Edit action without triggering the Details modal.
- Cattle ID validation against Master Cattle.
- Lookup supports:
  - Tag Number
  - Internal ID
  - Govt UID
  - Previous Tag Numbers
  - Historical numeric IDs where a leading zero was removed
- New Clinical Records can be added only for valid Active cattle.
- Existing historical records can be edited when their original cattle ID is unchanged.
- Read-only cattle summary displayed during Add/Edit and in the Details modal.
- Searchable checkbox selection for symptoms/diseases.
- Searchable checkbox selection for medicines.
- Doctor suggestions based on existing records.
- Multiline remarks and treatment instructions.
- Standard saving, success and error toasts.
- Add and Edit buttons disabled while saving.
- Backend sequential transaction ID generation:
  - MED-00001
  - MED-00002
  - etc.
- `health_log.transaction_id` is generated by Apps Script; no ARRAYFORMULA is used.
- Cattle identifiers are hard-written as plain text to preserve leading zeroes.

Current limitation:

- Symptoms and diagnosis remain stored together in the existing `symptom` field.
- Medicine dosage, frequency, duration and route are not yet stored separately.
- Clinical assessment and follow-up workflow enhancements are deferred to a later sprint.
