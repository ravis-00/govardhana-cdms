### health_log

| Column | Purpose |
|---|---|
| transaction_id | Permanent clinical record ID generated by Apps Script in `MED-#####` format |
| treatment_date | Date of clinical visit |
| internal_id | Cattle identifier entered during treatment; stored as plain text to preserve leading zeroes |
| weight_at_treatment | Weight at visit, currently optional/not yet implemented in the UI |
| symptom | Selected disease/symptom values stored as comma-separated text |
| diagnosis_detail | Existing diagnosis-detail field; richer clinical assessment is deferred |
| medicine_given | Selected medicines stored as comma-separated text |
| dosage_details | Existing field; structured dosage workflow is deferred |
| doctor_name | Treating doctor/veterinarian |
| treatment_cost | Existing optional field |
| photo_url | Treatment attachment/photo URL |
| remarks | Clinical remarks and treatment instructions |

Rules:

- `transaction_id` must be unique.
- IDs are generated by backend code; no spreadsheet formula is used.
- Valid sequential IDs follow `MED-#####`.
- New Clinical Records require a valid Active cattle record.
- Cattle identifiers must be written as plain text.
